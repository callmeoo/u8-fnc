# BMS-批量结算保存草稿交互文档

**版本**：v0.1  
**日期**：2026-06-04  
**范围**：BMS > 报销核算中心 > 批量结算  
**说明**：本文只整理交互逻辑，不涉及功能实现。

---

## 一、背景与目标

当前业务员在 BMS 批量结算中，从「待提交」车辆明细列表勾选车辆后，点击「马上申请结算」进入填写申请表单。现有 PRD 提到「保存草稿」按钮，但只描述为“保存填写内容，状态保持待提交”，没有说明以下问题：

- 已勾选车辆是否仍出现在「待提交」列表中。
- 保存后的申请单在 BMS 批量结算列表中如何查询、继续编辑、提交或废弃。
- 保存草稿后是否允许其他人或同一人重复勾选同一车辆再次发起申请。

本次交互目标：

- 在填写申请表单页提供「保存草稿」能力。
- 在 BMS 批量结算列表新增「草稿」状态，让业务员能找回未提交的申请单。
- 草稿关联的车辆需要形成可见的车辆列表，并从可申请的「待提交」车辆池中临时占用，避免重复申请。
- 草稿不进入审批流，不触发企业微信审批通知，不进入 FNC 财务处理。

---

## 二、核心口径

### 2.1 状态口径

建议新增独立的申请单状态「草稿」，不要继续用「待提交」表示已保存草稿。

| 对象 | 状态 | 含义 | 是否可被再次勾选申请 |
|---|---|---|---|
| 车辆明细 | 待提交 | 明细已进入待结算池，尚未被任何申请单占用 | 是 |
| 车辆明细 | 草稿占用 | 明细已被某张草稿申请单关联，尚未提交审批 | 否 |
| 车辆明细 | 审批中/已提交结算 | 明细已随申请单提交审批或后续流程 | 否 |
| 申请单 | 草稿 | 业务员已保存申请单，未提交审批 | 不适用 |
| 申请单 | 待审批/待支付/已结算/已驳回/已废弃 | 沿用现有流程 | 不适用 |

原因：如果草稿仍保持「待提交」，列表上无法区分“完全未发起申请的车辆”和“已填写一半的申请单”，也无法避免重复发起。

### 2.2 页面口径

BMS 批量结算列表按两类展示：

| Tab | 展示对象 | 说明 |
|---|---|---|
| 待提交 | 车辆明细列表 | 只展示可勾选发起申请的车辆，不展示已被草稿占用的车辆 |
| 草稿 | 申请单列表，支持查看关联车辆 | 已确认按申请单列表展示；展示当前用户保存的草稿申请单，进入详情或编辑页可查看车辆列表 |
| 待审批/待支付/已结算/已驳回/已废弃 | 申请单列表 | 沿用现有申请单列表 |
| 全部 | 申请单列表 | 建议包含草稿申请单，但仍不混入「待提交」车辆明细 |

「草稿」Tab 不直接平铺车辆明细，车辆明细通过草稿申请单的「查看车辆」或编辑页「已选明细」查看。

## 三、入口与按钮

### 3.1 待提交车辆列表

入口：BMS > 批量结算 >「待提交」Tab。

用户操作：

1. 业务员勾选车辆。
2. 系统沿用现有校验：
   - 不允许跨 U8 费用类型。
   - 服务费按同一集团归并。
   - 过户费、整理费-牌证及邮递费等其他费用按同一服务中心归并。
3. 点击右上角「马上申请结算」。
4. 进入填写申请表单页。

「待提交」Tab 不展示已被草稿占用的车辆。可在筛选区或说明文案补充：“已保存为草稿的车辆请到「草稿」Tab 查看。”

### 3.2 填写申请表单页

按钮建议调整为：

| 按钮 | 位置 | 行为 |
|---|---|---|
| 提交审批 | 主按钮 | 校验全部必填项，通过后提交审批 |
| 保存草稿 | 次按钮 | 保存当前已填内容和已选车辆，不提交审批 |
| 继续添加车辆 | 次按钮 | 返回待提交列表追加车辆，草稿已选车辆保持勾选或以“已选”区展示 |
| 取消/返回 | 次按钮 | 未保存内容时提示是否放弃 |

保存草稿允许必填项不完整，但必须满足：

- 至少有 1 条车辆明细。
- 已选车辆仍满足归并规则。
- U8 费用类型已由车辆带入。

保存草稿不要求发票、付款方、收款方、付款摘要、U8 部门等字段齐全。

## 四、保存草稿流程

### 4.1 首次保存草稿

流程：

1. 用户在填写申请表单页点击「保存草稿」。
2. 系统创建一张草稿申请单，生成草稿编号，例如 `BSD202606001`。
3. 系统保存：
   - 已选车辆明细 ID。
   - 标题、U8 费用类型、付款方、付款种类、收款方/发单人、付款摘要月份、发票类型、附件、备注等已填字段。
   - 申请人、保存时间、最后更新时间。
4. 底层车辆状态从「待提交」变为「草稿占用」。
5. 页面提示：“草稿已保存，可在「草稿」Tab 继续编辑或提交审批。”
6. 页面跳转建议：
   - 默认跳回 BMS 批量结算列表「草稿」Tab，并高亮刚保存的草稿。
   - 如果用户点击的是表单内自动保存，可停留在当前页。

### 4.2 编辑已有草稿后再次保存

流程：

1. 用户从「草稿」Tab 点击「编辑」进入申请表单。
2. 修改字段或车辆明细。
3. 点击「保存草稿」。
4. 系统更新原草稿，不生成新草稿编号。
5. 若新增车辆，则新增车辆进入「草稿占用」。
6. 若移除车辆，则被移除车辆恢复为「待提交」。
7. 更新最后保存时间。

### 4.3 草稿提交审批

流程：

1. 用户从草稿编辑页点击「提交审批」。
2. 系统按正式提交规则校验全部必填项。
3. 校验通过后：
   - 申请单状态从「草稿」变为「待审批」。
   - 底层车辆状态从「草稿占用」变为「审批中/已提交结算」。
   - 进入现有审批流。
   - 触发企业微信和 BMS 站内审批通知。
4. 校验不通过时，停留在表单页并定位到缺失字段。

## 五、BMS 批量结算列表「草稿」Tab

### 5.1 列表字段

「草稿」Tab 已确认展示申请单列表，而不是平铺所有车辆。

| 字段 | 说明 |
|---|---|
| 草稿编号 | 系统生成，保存草稿时产生 |
| U8 费用类型 | 由车辆明细带入，只读 |
| 标题 | 用户已填写则展示；未填写展示“未填写” |
| 付款种类 | 对外支付/冲销借支；未填写展示“未填写” |
| 付款摘要 | 已生成则展示；未填写展示“未填写” |
| 明细数 | 草稿关联车辆数量 |
| 明细合计 | 草稿关联车辆含税金额合计 |
| 归并维度 | 服务费展示集团；其他费用展示服务中心 |
| 申请人 | 草稿创建人 |
| 最后保存时间 | 最近一次保存时间 |
| 操作 | 编辑 / 查看车辆 / 废弃 |

### 5.2 筛选项

建议支持：

- 保存时间。
- U8 费用类型。
- 付款种类。
- 标题。
- 草稿编号。
- 集团。
- 服务中心。
- 申请人。

普通业务员默认只看自己的草稿；主管或管理员是否可看全部草稿需要权限确认。

### 5.3 操作项

| 操作 | 行为 |
|---|---|
| 编辑 | 进入填写申请表单页，回显草稿字段和车辆明细 |
| 查看车辆 | 弹窗或详情页展示草稿关联车辆列表 |
| 废弃 | 二次确认后草稿状态变「已废弃」，底层车辆恢复「待提交」 |

废弃确认文案建议：

> 废弃后该草稿不可恢复，已占用的车辆将释放回「待提交」列表，可重新勾选发起申请。确认废弃？

## 六、草稿车辆列表

草稿车辆列表用于回答“草稿状态下有哪些车辆被占用”。

### 6.1 展示入口

入口一：BMS 批量结算列表 >「草稿」Tab > 某草稿行 >「查看车辆」。  
入口二：草稿编辑页底部「已选明细」区。

### 6.2 展示字段

| 字段 | 说明 |
|---|---|
| 车牌号 | 来自底层明细 |
| VIN码 | 来自底层明细 |
| 清分时间 | 来自底层明细 |
| 结算机构 | 来自底层明细 |
| 费用类型 | 业务费用类型 |
| U8 费用类型 | 服务费/过户费/整理费-牌证及邮递费 |
| 集团 | 服务费归并维度 |
| 门店 | 原有字段 |
| 服务中心 | 过户费、整理费归并维度 |
| 实际结算金额含税 | 金额 |
| 明细状态 | 草稿占用 |

### 6.3 车辆释放规则

| 场景 | 车辆状态变化 |
|---|---|
| 保存草稿 | 待提交 → 草稿占用 |
| 草稿中移除车辆 | 草稿占用 → 待提交 |
| 草稿废弃 | 草稿占用 → 待提交 |
| 草稿提交审批 | 草稿占用 → 审批中/已提交结算 |
| 审批驳回后废弃 | 按现有规则恢复待提交 |
| 财务打回后 | 按现有规则恢复待提交 |

## 七、继续添加车辆

### 7.1 从新申请表单继续添加

用户点击「继续添加车辆」后：

1. 返回「待提交」车辆列表。
2. 已选车辆保留在“已选明细”中，并在列表中以“已选”状态展示。
3. 用户可继续勾选同组车辆。
4. 系统继续执行同一 U8 费用类型和归并维度校验。
5. 用户再次点击「马上申请结算」回到同一张临时申请表单。

此时如果用户未点击「保存草稿」，车辆可以只在前端临时选中，不需要进入「草稿占用」。

### 7.2 从已有草稿继续添加

用户从草稿编辑页点击「继续添加车辆」后：

1. 返回「待提交」车辆列表。
2. 当前草稿已关联车辆不出现在可勾选列表，或展示为“已在本草稿中”且不可重复勾选。
3. 列表只允许选择与该草稿同一 U8 费用类型、同一归并维度的车辆。
4. 新增后回到草稿编辑页。
5. 新增车辆在再次保存草稿后进入「草稿占用」；若采用实时保存，则添加时即进入「草稿占用」。

建议采用“添加后立即更新草稿占用”，减少用户关闭页面导致选择丢失的风险。

## 八、校验与提示

### 8.1 保存草稿校验

保存草稿只做最低限度校验：

- 未选择车辆：提示“请先选择需要结算的车辆”。
- 混选 U8 费用类型：提示“所选车辆须属于同一 U8 费用类型，当前混选了：{费用类型}”。
- 服务费跨集团：提示“服务费须按同一集团归并，当前混选了：{集团}”。
- 其他费用跨服务中心：提示“该费用类型须按同一服务中心归并，当前混选了：{服务中心}”。
- 草稿关联车辆已被其他申请占用：提示“部分车辆状态已变化，请刷新后重新选择”。

### 8.2 提交审批校验

提交审批沿用正式申请校验：

- 标题必填。
- U8 费用类型必填且只读。
- 付款方、U8 部门、付款种类、收款方/发单人、付款摘要月份、发票类型等按现有规则必填。
- 车辆明细至少 1 条。
- 明细金额合计与付款金额一致。

### 8.3 离开页面提示

用户在表单有未保存修改时点击返回、切换菜单或关闭页面：

> 当前申请表单存在未保存内容，是否保存为草稿？

按钮：

- 保存草稿：执行保存并跳转。
- 不保存离开：放弃本次未保存内容；如果是新申请且未保存草稿，车辆不占用。
- 继续编辑：关闭提示，停留当前页。

## 九、权限规则

| 角色 | 草稿权限 |
|---|---|
| 业务员 | 创建、编辑、提交、废弃自己的草稿 |
| 部门负责人 | 默认不可审批草稿；是否可查看下属草稿待确认 |
| 财务人员 | 默认不可处理草稿；草稿不进入 FNC |
| 系统管理员 | 可按管理权限查询草稿，是否可废弃他人草稿待确认 |

草稿不触发审批通知，不出现在 FNC 批量结算待审批、待支付、凭证来源记录中。

## 十、异常场景

### 10.1 草稿车辆被底层数据调整

如果草稿关联的车辆金额、费用类型、集团、服务中心等关键字段在保存后被系统任务或管理员调整：

- 用户打开草稿时，系统提示“草稿关联明细发生变化，请核对后再提交”。
- 金额变化时重新计算合计。
- U8 费用类型或归并维度变化导致不满足规则时，禁止提交审批，要求移除异常车辆或重新拆分申请。

### 10.2 草稿长期未处理

建议设置草稿有效期或清理机制，例如 30 天未更新的草稿提醒创建人处理。是否自动释放车辆需业务确认，默认不自动释放，避免用户已准备好的申请被系统误清理。

### 10.3 同一车辆重复保存草稿

后端保存时需要以车辆明细 ID 做占用校验。同一车辆同一时间只能归属一个未终态申请单，包括草稿、待审批、待支付等状态。

## 十一、与现有文档的差异建议

现有 PRD 中「保存草稿」描述为：

> 保存填写内容，状态保持“待提交”，不发起审批

建议调整为：

> 保存填写内容并生成“草稿”申请单，不发起审批；草稿关联的车辆从“待提交”车辆池中临时占用，在草稿提交审批、移除车辆或废弃草稿时按规则流转。

同时建议在 BMS 批量结算列表状态枚举中新增：

- 草稿。

并在「待提交」Tab 说明中补充：

- 「待提交」仅展示可发起申请的车辆明细。
- 已保存为草稿的车辆不在「待提交」中展示，请到「草稿」Tab 查看和继续处理。

---

## 十二、待确认问题

1. 草稿是否只允许创建人本人查看和编辑？部门负责人、财务、管理员是否需要查询入口？
2. 草稿是否需要有效期，例如超过 30 天未更新提醒或自动释放？
3. 新申请未保存时点击「继续添加车辆」，是否需要先自动保存草稿，还是仅保留临时选择？本文建议新申请不自动占用，已有草稿添加车辆时实时占用。
4. 保存草稿时附件是否立即上传并持久化？本文建议保存草稿时已选择附件同步保存，废弃草稿后附件按文件生命周期策略处理。
5. 已保存草稿是否需要支持复制新建？本文暂不纳入，避免重复占用车辆。

---

## 十三、验收标准

- BMS 批量结算列表出现「草稿」状态 Tab。
- 用户在填写申请表单页可点击「保存草稿」并生成草稿申请单。
- 草稿可在「草稿」Tab 查询、编辑、查看车辆、废弃。
- 草稿关联车辆不再出现在可勾选的「待提交」车辆列表中。
- 草稿废弃或移除车辆后，相关车辆恢复到「待提交」。
- 草稿提交审批后进入现有审批流，状态变为「待审批」。
- 草稿不出现在 FNC 待审批、待支付、凭证来源记录中。
- 草稿保存不触发企业微信审批通知。
- 混选 U8 费用类型、跨集团/跨服务中心等规则在保存草稿和提交审批时均生效。
