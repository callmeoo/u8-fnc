## 一、背景与目标

### 1.1 系统背景说明

批量结算功能在 **BMS**（业务后台）和 **FNC**（财务后台）中均有对应模块，两者共享同一套数据：

- **业务员**：在BMS中发起申请、上传发票、提交审批
- **财务人员**：在FNC中查看申请数据、审核核对、操作凭证
- **审批流程**：同时支持PC端和移动端审批

本需求的操作入口在 **FNC财务系统**，无需额外的跨系统数据同步。

### 1.2 现状痛点

批量结算申请完成三级审批后，财务人员需在FNC"凭证管理"中**手动逐笔新增凭证并填写分录**，再点击"开具"推送至U8。每月约 **50个凭证、11100条明细**，数据量大、手工填写易出错。

### 1.3 目标

在FNC"凭证管理"模块新增"从批量结算导入"能力：财务选择已审批通过的BMS结算申请，系统**自动生成凭证分录**，财务确认后一键推送U8，无需手动录入。

> BMS审批通过（待支付）→ 财务在FNC从批量结算新增凭证 → 系统自动填充分录 → 财务确认 → 点击开具 → 推送U8 → 凭证号回写

### 1.4 涉及范围

- **涉及系统**：FNC财务系统、U8（用友）
- **涉及模块**：FNC > 财务管理 > 凭证管理
- **费用类型**：服务商服务费、代办过户费、牌证费及邮寄费（整理费）
- **涉及角色**：财务人员、系统管理员
- **联动需求一**：BMS申请表单"结算说明"字段改造（见4.5）
- **不涉及**：BMS/FNC审批流程改造（见需求一）、U8系统内部配置、付款执行

## 四、凭证生成规则

### 4.1 颗粒度

| 规则 | 说明 |
|------|------|
| 一次操作（可选多笔BMS申请）= 一张U8凭证 | 多笔申请的明细合并进同一张凭证，不拆分 |
| 每条明细 = 一条借方费用分录 | 按车牌号逐条生成，多笔申请的明细依次排列 |
| 进项税额汇总为一行 | 所有明细的税额加总，不按申请或明细拆分 |
| 银行存款汇总为一行 | 所有申请的含税合计加总，贷方一行 |

### 4.2 会计分录结构

| 序号 | 科目编码 | 科目名称 | 借/贷 | 金额 | 辅助核算 |
|------|---------|---------|------|------|---------|
| 1…N | 见4.3 | 费用科目 | 借 | 各车不含税金额（见4.2.1计算规则） | 自定义项1 = 车牌号；整理费需填部门编码 |
| N+1 | 2221010101 | 进项税额 | 借 | Σ 各车进项税额 | — |
| N+2 | 10020101 | 银行存款 | 贷 | 含税合计（Σ 各车含税结算金额） | — |

#### 4.2.1 逐车进项税额计算规则

**税率由财务人员在生成凭证时选择**（下拉选择，选项由管理员在后台配置），系统根据所选税率自动计算，不硬编码。

系统对每条明细（每台车）**单独计算**进项税额和不含税金额：

```
单车进项税额   = round( 该车含税结算金额 ÷ (1 + 税率) × 税率, 2 )  ← 四舍五入保留2位小数
单车不含税金额 = 该车含税结算金额 - 单车进项税额
```

**示例**（财务选择税率6%，5台车每台含税结算金额200元）：

| 车辆 | 含税结算金额 | 进项税额 | 不含税金额 | 验证 |
|------|-----------|--------|---------|------|
| 车1 | 200.00 | 200÷1.06×6% = **11.32** | 200-11.32 = **188.68** | 11.32+188.68=200 ✓ |
| 车2 | 200.00 | **11.32** | **188.68** | ✓ |
| 车3 | 200.00 | **11.32** | **188.68** | ✓ |
| 车4 | 200.00 | **11.32** | **188.68** | ✓ |
| 车5 | 200.00 | **11.32** | **188.68** | ✓ |
| **汇总** | **1000.00** | **56.60** | **943.40** | 56.60+943.40=1000 ✓ |

凭证分录：
- 5条费用借方行，各188.68元
- 进项税额借方汇总行：56.60元
- 银行存款贷方汇总行：1000.00元

> **借贷平衡校验**：Σ不含税金额 + Σ进项税额 = 含税合计（即银行贷方），系统在"开具"推送前自动校验，不平衡则阻断并提示差额。

### 4.3 费用科目映射

| BMS费用项目 | U8费用项目 | 科目编码 | 是否需要部门核算 |
|------------|----------|---------|--------------|
| 服务商服务费 | 服务费 | 64010102 | 否 |
| 代办过户费 | 过户费 | 64010103 | 否 |
| 牌证费及邮寄费 | 整理费 | 64011209 | **是** |

> 科目映射关系由系统管理员在后台维护，不硬编码，支持后续调整（见第六章）。

### 4.4 整理费部门核算

整理费（64011209）的每条借方分录需填写U8部门编码。

> **✅ 已确认采用方案二**：U8无可用部门查询接口。

**方案二（已采用）：后台维护U8部门字典**

> ⚠️ **重要修订**：结算机构与U8部门编码**没有直接映射关系**，原"结算机构 → U8部门编码"映射表方案已废弃。

- 系统管理员在FNC后台维护一份**U8部门字典**（U8部门编码 + U8部门名称），路径：
  - FNC后台管理 > 基础配置 > U8部门编码维护，或
  - FNC > 财务管理 > 凭证管理 > "部门编码映射" tab
- 字典初始化数据参考U8系统中实际启用的部门列表（如：66 总经办、7402 车源生态事业部（武汉）、7403 车源生态事业部（长沙）、01 财务部 等，详见原型 deptMappingData）。
- 财务新增凭证时，系统**不再根据结算机构自动带入**部门编码；财务在凭证预览弹窗中通过**下拉选择**U8部门，所选编码会同步更新所有借方分录行（允许逐行覆盖）。
- 部门下拉的可选项来源于上述U8部门字典中"启用"状态的记录。

部门编码为**必填项**，未填写不允许确认创建凭证。

### 4.5 凭证摘要格式

**格式**：`付{月份}{费用项}，{收款公司全称}`

**示例**：`付1月服务费，常有（天津）信息技术有限公司`

字段来源：

| 摘要组成 | 来源 |
|---------|------|
| 月份 | BMS申请表单"付款月份"下拉框（见4.5.1） |
| 费用项 | BMS费用类型映射的U8费用项目名称（如"服务费"） |
| 收款公司全称 | BMS申请表单"付款信息"区块中的"**收款方名称**"字段（见需求一PRD 5.1.2，与4.5.1中"付款月份"联动，统一不再单设"收款单位"字段） |

**多笔申请合并时的摘要规则**：

> ⚠️ **重要约束（见5.2.1）**：FNC选择来源记录时，**多选的申请必须属于同一收款方**（"收款方名称"相同），系统在点击"下一步"时强制校验，不同收款方的申请不允许合并为同一张凭证。此约束从源头消除了"银行存款行摘要语义不明"的问题（如一张凭证同时包含A公司和B公司款项时，银行存款行摘要无法如实反映）。

在同一收款方前提下的摘要规则：
- **费用分录行**：每笔申请的分录行各自使用本申请的"收款方名称"和"付款月份"拼接摘要（多笔月份可不同，属正常合并场景）
- **进项税额汇总行**：摘要统一为`付{第一笔申请月份}{费用项}，{收款方名称}`
- **银行存款汇总行**：摘要统一为`付{第一笔申请月份}{费用项}，{收款方名称}`（因已强制同收款方，语义明确）

### 5.2 从批量结算新增凭证

#### 5.2.1 第一步：选择来源记录

点击"从批量结算新增"，弹出选择面板，包含以下筛选和列表：

**筛选条件**：
- 费用项类型（必选）：服务商服务费 / 代办过户费 / 牌证费及邮寄费
- 结算年月（必选）：月份选择器

**记录列表**（按筛选条件展示BMS待支付申请）：

| 字段 | 说明 |
|------|------|
| 结算申请编号 | BMS申请编号 |
| 结算机构 | 来自BMS |
| 收款方名称 | 来自BMS付款信息（即需求一5.1.2"收款方名称"字段） |
| 含税合计（元） | 来自BMS |
| 明细条数 | 该申请包含的车辆记录数 |
| BMS申请时间 | — |

支持**多选**，一次可勾选多笔申请，所有选中记录合并生成**一张凭证**。

**多选校验规则**（点击"下一步"时触发）：

1. **防重复凭证**：若勾选的申请中有任意一笔已存在未取消凭证，**整批阻断**，提示冲突申请编号，财务取消勾选该笔后方可继续。

2. **收款方一致性**（新增）：**多选的申请必须属于同一收款方（"收款方名称"字段值相同）**，否则整批阻断，提示"所选申请包含不同收款方，请分批生成凭证"。此规则确保凭证摘要（尤其是银行存款汇总行）语义明确，符合财务规范——若允许混合不同收款方，银行存款行将无法如实反映"付给谁"。

点击"下一步"进入凭证预览。

#### 5.2.2 第二步：凭证分录预览与确认

系统根据所选申请自动生成分录，展示预览页面：

| 区块 | 展示内容 |
|------|---------|
| 基本信息 | 结算申请编号、结算机构、结算年月、收款方、含税合计 |
| 凭证信息 | 凭证类别（记）、制单日期（默认今日，可修改）、制单人（当前登录用户）、**账套（下拉必选，对应U8主体）**、**进项税率（下拉必选，选后自动重算所有分录进项税额和不含税金额）** |
| 分录明细 | 行号、借/贷、科目编码、科目名称、摘要、**含税金额**、**进项税额**、借方金额（不含税）、贷方金额、车牌号（自定义项1）、部门编码 |
| 借贷合计 | 借方合计 / 贷方合计，实时显示是否平衡 |

> 分录明细中"含税金额"和"进项税额"为 FNC 内部展示字段（不推送至U8），方便财务核对计算逻辑；推送U8的借方金额为"不含税金额"。

**整理费特殊处理**：含整理费分录时，部门编码列显示为必填下拉框，未填写不允许确认。

**操作按钮**：取消 / 确认

点击"确认"后：
- 系统校验借贷是否平衡，不平衡提示差额，阻断
- 校验通过，凭证记录创建，状态变为**待开具**，返回凭证列表

---

### 5.3 开具（推送U8）

财务在凭证列表中找到"待开具"状态的凭证，点击**"开具"**，系统调用U8 API推送凭证数据。

| 结果 | 状态变更 | 说明 |
|------|---------|------|
| 推送成功 | 待开具 → 已开具 | U8凭证号回写FNC，"开具"按钮置灰 |
| 推送中 | 待开具 → 开具中 | 按钮置灰，防止重复点击 |
| 推送失败 | 待开具 → 开具失败 | 失败原因记录，可通过hover tooltip查看 |

---

### 5.4 开具失败处理

与现有FNC凭证管理"开具失败"Tab保持一致：

- **查看失败原因**：鼠标悬停于该行，tooltip展示失败时间和原因（如：会计期间早于开账日期、科目不存在等）
- **重试**：点击"开具"按钮，使用已保存的凭证分录数据重新推送U8
- **取消**：状态变为"已取消"，不可恢复

### 5.5 U8接口对接

**现有实现**：FNC系统已有完整的U8对接工具类（`U8HttpClient`），通过BIP中间件与U8通信，无需重新开发底层接口，本需求在已有框架上扩展。

#### 5.5.1 推送凭证接口

- **接口**：`POST {u8_base_url}/api/bip/voucher/add`
- **请求结构**：

```json
{
  "from_account": "WP",
  "to_account": "U8",
  "ds_sequence": "账套编码",
  "voucher": {
    "csign": "记",
    "fiscal_year": "2026",
    "accounting_period": "1",
    "date": "2026-01-15",
    "enter": "制单人姓名",
    "coutsysname": "WP",
    "coutsign": "业务系统凭证",
    "coutid": "系统流水号",
    "attachment_number": 5,
    "entry": [
      {
        "entry_id": 1,
        "cdigest": "付1月服务费，常有（天津）信息技术有限公司",
        "cexch_name": "人民币",
        "nfrat": 1,
        "ccode": "64010102",
        "md": 188.68,
        "mc": 0
      }
    ]
  }
}
```

> 辅助核算（clientAuxiliary）：批量结算凭证暂不需要供应商辅助核算，现金流量项目是否需要待技术侧与财务确认。

#### 5.5.2 凭证字段来源（批量结算场景）

| U8字段 | 来源 |
|--------|------|
| csign | 固定值：记 |
| fiscal_year | BMS结算年月的年份 |
| accounting_period | BMS结算年月的月份 |
| date | 财务确认凭证的当日 |
| enter | 优先查`u8_issue_user`配置表（FNC用户ID→U8姓名映射）；无配置则取FNC登录用户名 |
| ds_sequence | 财务在凭证预览时选择的账套编码（见5.5.3） |
| attachment_number | 本张凭证分录总行数 |
| entry[].ccode | 费用科目编码（见4.3科目映射） |
| entry[].md | 费用行：不含税金额；税额行：进项税额汇总 |
| entry[].mc | 银行存款行：含税合计 |
| entry[].cdigest | 付{月份}{费用项}，{收款公司全称} |

#### 5.5.3 账套（dsSequence）

支持多主体账套（唯普WP / 唯普云WPY / 机拍JP等），**根据结算机构对应不同主体**，由财务人员在凭证预览时**从下拉列表选择**。账套选项由管理员在FNC后台配置维护。

#### 5.5.4 U8返回值处理

| 返回情况 | 处理逻辑 |
|---------|---------|
| errcode="0" | 成功：voucherNo = "记" + ino_id，回写FNC，状态→已开具 |
| errcode="30115" | 凭证已存在，视为成功，状态→已开具 |
| 其他errcode | 失败：记录errcode和errmsg，状态→开具失败，错误信息可在列表tooltip查看；点击"开具"可重试（U8明确返回失败，无重复入账风险） |
| **网络超时 / 无响应码** | ⚠️ **高风险**：超时时U8可能已接收并落账，若直接标记"失败"允许重试，将导致U8重复生成凭证。**处理方式**：状态→"**开具超时**"（新增状态），凭证列表显示告警提示"推送超时，请先在U8系统核查是否已生成凭证，再选择操作"；提供两个操作入口：①"**手动回填凭证号**"（财务在U8确认已生成后直接填入凭证号，状态→已开具）；②"**重新开具**"（财务在U8确认**未**生成后方可点击，状态回到待开具后重新推送）。超时阈值建议由技术侧结合U8接口SLA确认（见待确认事项P6）。 |

#### 5.5.5 部门查询接口（整理费）

> ⚠️ **待确认（P2）**：U8是否有可用的部门查询接口？影响整理费分录部门编码的选择方式（见4.4两种方案）。

### 5.6 财务打回（逆向退回至BMS）

#### 5.6.1 触发场景

BMS申请已通过三级审批（状态"待支付"），财务在FNC查看凭证预览时发现信息有误（如收款方名称错误导致摘要拼接不对、付款项目说明有误等），此时申请已无法由业务员自行废弃，需通过FNC发起"打回"操作将其退回业务员。

#### 5.6.2 操作入口与可见权限

| 入口 | 触发条件 | 可见角色 |
|------|---------|---------|
| FNC凭证管理列表 > 凭证行 > "打回"按钮 | 凭证状态为"**待开具**"（已创建凭证但尚未推U8） | 财务人员 |
| BMS批量结算列表 > 申请行 > "打回"按钮 | BMS申请状态为"**待支付**" | 财务人员 |

> **约束**：已推送U8成功（"已开具"）的凭证**不支持打回**，须在U8侧操作反记账，超出本系统能力范围。

#### 5.6.3 打回执行逻辑

1. 财务点击"打回"，弹出原因输入框（必填，不超过200字）
2. 确认后系统执行：
   - BMS申请状态：**待支付 → 已驳回**（驳回来源标记"财务打回"，显示打回人姓名和时间）
   - FNC凭证（如已创建"待开具"状态）：同步变为"**已取消**"
   - 申请关联的所有底层明细：恢复为"**待提交**"状态
3. 通知申请人（企业微信 + BMS站内消息）：  
   `"您的结算申请已被财务打回，请修改后重新提交，原因：{打回原因}"`
4. 业务员在"已驳回"状态下修改相关信息后，可重新发起三级审批流程

## 六、科目映射后台配置

**路径**：FNC后台管理 > 基础配置 > 凭证科目映射

| 配置项 | 说明 |
|--------|------|
| BMS费用项目 | 如：服务商服务费 |

| U8科目名称 | 如：服务费 |

**税率配置**（独立维护）：管理员在后台维护可选税率列表（如：0%、3%、6%、9%、13%等），财务生成凭证时从下拉框选择，系统据此计算各车进项税额。同一张凭证使用统一税率。

系统管理员可在此增删改查，修改后新生成凭证时生效，历史凭证不受影响。
